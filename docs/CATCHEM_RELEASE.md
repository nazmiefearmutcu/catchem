# Catchem — Release Build & macOS Distribution

## Build artifacts

| Artifact | Path | Built by |
|---|---|---|
| `fusion-stack-sidecar/` | `desktop/catchem/sidecar-out/` | `build_sidecar.sh` (PyInstaller --onedir) |
| `Catchem.app` | `desktop/catchem/src-tauri/target/release/bundle/macos/` | `build_catchem_release.sh` |
| `Catchem_<ver>_<arch>.dmg` | `desktop/catchem/src-tauri/target/release/bundle/dmg/` | same (Tauri produces it inline) |

## One-shot release

```bash
cd /Users/nazmi/Desktop/Projeler/proje/fusion_stack
bash desktop/catchem/scripts/build_catchem_release.sh
```

This runs in order:

1. Build the React bundle into `src/fusion_stack/static/app/`.
2. Build the Catchem boot shim.
3. PyInstaller `--onedir` packages the sidecar into a self-contained binary.
4. Stage the sidecar under `src-tauri/resources/sidecar/`.
5. `cargo tauri build` produces `Catchem.app` + `Catchem_<ver>.dmg`.

The release build takes ~5–8 minutes on Apple Silicon (the bulk is the
PyInstaller step + the linker for the static Rust binary).

## Signing

`build_catchem_release.sh` reads four env vars. If unset, the build still
succeeds but produces an unsigned `.app`.

```bash
export APPLE_DEVELOPER_IDENTITY="Developer ID Application: Your Name (TEAMID)"
export APPLE_ID=you@example.com
export APPLE_TEAM_ID=TEAMID
export APPLE_APP_SPECIFIC_PASSWORD=xxxx-xxxx-xxxx-xxxx
```

The first var enables codesigning via the Tauri macOS bundler. The other
three enable post-build notarization via `notarytool`.

To find your signing identity:

```bash
security find-identity -p codesigning -v
```

## Notarization

`build_catchem_release.sh` automatically submits the produced `.dmg` to
Apple's notarization service when all three notarization env vars are set,
then staples the result.

Manual run (if you skipped it during build):

```bash
DMG=desktop/catchem/src-tauri/target/release/bundle/dmg/Catchem_0.1.0_aarch64.dmg
xcrun notarytool submit "$DMG" \
  --apple-id "$APPLE_ID" \
  --team-id "$APPLE_TEAM_ID" \
  --password "$APPLE_APP_SPECIFIC_PASSWORD" \
  --wait
xcrun stapler staple "$DMG"
```

## Installing the unsigned dev/QA build

If you build without `APPLE_DEVELOPER_IDENTITY`, macOS will refuse to open
the `.app` from a download (Gatekeeper). Two ways to bypass for QA:

```bash
# Single-shot: remove the quarantine attribute
xattr -dr com.apple.quarantine /Applications/Catchem.app

# Or, on first launch:
#   Right-click Catchem.app → Open → confirm in the warning dialog
```

These are dev-only workarounds. Distributing to other users requires a
signed + notarized build.

## Where Catchem stores data

| Path | What lives there |
|---|---|
| `~/Desktop/Projeler/proje/fusion_stack/data/` (dev) | SQLite, parquet exports, vector index, demo input |
| `~/Library/Application Support/Catchem/` (release) | (future) Catchem user preferences |
| `~/Library/Logs/Catchem/` (release) | (future) Catchem-specific logs |
| `data/logs/api.out` | sidecar stdout/stderr |
| `data/logs/fusion_stack.log` | structured log surfaced via `/ui/log-tail` |

The release build's sidecar should be reconfigured to write to
`~/Library/Application Support/Catchem/data/` instead of the repo dir —
this is tracked as a follow-up.

## Replacing placeholder icons

`desktop/catchem/src-tauri/icons/` ships with solid-color PNGs generated by
`build_catchem_dev.sh`. Before any release:

```bash
# Use the official Apple `iconutil` to build a real .icns
mkdir my.iconset
sips -z 16 16     icon-1024.png --out my.iconset/icon_16x16.png
sips -z 32 32     icon-1024.png --out my.iconset/icon_16x16@2x.png
sips -z 32 32     icon-1024.png --out my.iconset/icon_32x32.png
sips -z 64 64     icon-1024.png --out my.iconset/icon_32x32@2x.png
sips -z 128 128   icon-1024.png --out my.iconset/icon_128x128.png
sips -z 256 256   icon-1024.png --out my.iconset/icon_128x128@2x.png
sips -z 256 256   icon-1024.png --out my.iconset/icon_256x256.png
sips -z 512 512   icon-1024.png --out my.iconset/icon_256x256@2x.png
sips -z 512 512   icon-1024.png --out my.iconset/icon_512x512.png
cp icon-1024.png             my.iconset/icon_512x512@2x.png
iconutil -c icns my.iconset -o desktop/catchem/src-tauri/icons/icon.icns
```

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| Boot screen stuck on "Starting local sidecar…" 30s | Sidecar crashed | `tail -120 data/logs/api.out`; check Python is in PATH; restart from menu |
| Catchem opens but UI shows "bundle not built" placeholder | React bundle missing | `(cd frontend && npm install && npm run build)` |
| "rejected unsafe external url" in console | A link in a record points at `javascript:` / `data:` | Expected — `safeHref` is doing its job |
| `cargo tauri build` fails with `Xcode not found` | Command line tools missing | `xcode-select --install` |
| Sidecar binary in `.app` rejects with "killed: 9" | Hardened runtime entitlements wrong | Re-sign with `--options runtime --entitlements src-tauri/Entitlements.plist` |
